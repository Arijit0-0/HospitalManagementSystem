# CareAxis Hospital Management System Report

## 1. Project Title

CareAxis Hospital Management System

## 2. Objective

The objective of this project is to create a web-based hospital management system that improves the handling of patient appointments, doctor records, user access, and administrative reporting in a centralized platform.

## 3. Problem Statement

Hospitals and clinics often manage appointments, patient information, and doctor assignments manually or using disconnected systems. This creates delays, poor visibility, and difficulty in maintaining records. A centralized management system helps reduce manual effort and improves operational control.

## 4. Proposed Solution

This project provides a role-based hospital management application where:

- Patients can register and book appointments.
- Doctors and administrators can manage records.
- Administrators can view reports and audit logs.
- The system stores all data in MySQL and uses Flask for application logic.

## 5. Modules Implemented

- User authentication
- Patient registration and appointment booking
- Doctor directory management
- Appointment editing and deletion
- Dashboard with real-time system counts
- Reports page with demand analysis
- Admin dashboard
- Audit log using MySQL triggers

## 8. Roles in the System

### Admin

- Access admin dashboard
- View reports
- Manage doctor registry
- View activity log
- View all bookings

### Doctor

- View appointments
- Access doctor-related workflow

### Patient

- Register account
- Book appointment
- View and edit personal bookings

## 9. Output Highlights

- Professional landing page
- Live dashboard metrics
- Department-wise doctor directory
- Appointment status labels
- Printable management reports
- Admin-only pages with controlled access

## 10. Conclusion

The CareAxis Hospital Management System successfully demonstrates how hospital operations can be digitized using Python Flask and MySQL. The project improves appointment handling, role-based access, and reporting while maintaining a clean user experience suitable for academic submission and live demonstration.
